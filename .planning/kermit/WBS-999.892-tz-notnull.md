# WBS — 999.892-tz-notnull — bugfix/hardening — 2026-06-26

## Intent
Adopt a single-source UTC contract for juggler. `users.timezone` is the per-user timezone
authority but is currently NULLABLE (`string(100) DEFAULT 'America/New_York'`, no `.notNullable()`),
so every reader guesses a `|| 'America/New_York'` fallback — a data-integrity hazard and a source
of the `new Date()` local-misparse bug class. Make the column NOT NULL DEFAULT 'America/New_York'
(backfilling existing NULLs first), then remove the now-redundant column-null fallbacks at reader
call sites WITHOUT weakening legitimate row-absence/DB-error guards.

Business acceptance: after migrate:latest, `users.timezone` is NOT NULL with default
'America/New_York'; any pre-existing NULLs are backfilled; readers no longer carry redundant
column-null fallbacks; migration reverses cleanly.

## Scooter consult
No prior Brain/scheduler-rules decision on the column constraint itself. Brain fact #77683 supports
the NOT NULL + DEFAULT 'America/New_York' direction. The A1 deliberate-null contract
(KnexConfigRepository.getUserTimezone) is recorded only in JSDoc, not a curated decision — so
reshaping it would need its own ruling (out of scope here). No veto found.
