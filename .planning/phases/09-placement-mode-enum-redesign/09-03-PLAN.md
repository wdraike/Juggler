<objective>
Update task.controller.js to remove derivePlacementMode() and write placement_mode directly from the client-supplied placementMode field.

Purpose: derivePlacementMode() inspects `when.includes('fixed')` — the root cause of the bug where user-defined tag names could cause misclassification. After this change, placement_mode is always written by the UI explicitly; the server never derives it from when-content.
Output: controller with simplified taskToRow placement block, corrected rowToTask fallback, cleaned PLACEMENT_TRIGGER_FIELDS, and fixed task-creation paths. takeOwnership dead-code filter removed.
</objective>
