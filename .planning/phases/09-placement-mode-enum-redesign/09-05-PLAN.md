<objective>
Update WhenSection.jsx and TaskEditForm.jsx to use placementMode as the single source of truth for scheduling mode. Remove the hasPreferredTime inference pattern. Extend the three-button mode selector to non-recurring tasks. Wire placementMode state through the form submit path.

Purpose: Without this change, the frontend continues to infer mode from `when` content and hasPreferredTime — the exact antipattern the backend now rejects. After this change, the UI writes placementMode explicitly and the backend stores it directly.
Output: UI where placementMode flows down from TaskEditForm state, mode buttons set it explicitly, and buildFields() includes it in every patch.
</objective>
